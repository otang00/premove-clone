# 07. 구현 순서

## 1순위: 공통 프레임
1. 프로젝트 기본 레이아웃
2. 헤더
3. 푸터
4. 공통 카드/버튼 스타일

## 2순위: 메인 예약 페이지 완성
1. 메인 비주얼 또는 안내 배너
2. 메인 예약 박스
3. 검색 조건 필드 구성
4. 검색 버튼과 이동 흐름 placeholder 연결
5. 보조 신뢰 영역 정리

## 3순위: 차량 목록 페이지 완성
1. 검색 조건 박스 재사용
2. 정렬 바
3. 차량 카드
4. 차량 리스트 전체 조립

## 4순위: 차량 상세/예약 페이지 완성
1. 상단 검색 조건 박스 재사용
2. 차량 요약 카드
3. 탭 영역
4. 보험 정보
5. 운전자 정보 폼
6. 대여 방법 선택
7. 업체 정보 + 지도 자리
8. 결제수단
9. 약관 동의
10. 우측 결제 박스

## 5순위: 연결/마감
1. 메인 검색 후 목록 이동
2. 목록 카드 클릭 시 상세 이동
3. 예약내역/FAQ placeholder 연결
4. 더미 상태값 정리
5. 여백/타이포/색상 마감

## 금지
- 1단계에서 실제 비즈니스 로직에 손대지 말 것
- 1단계에서 관리자 구조까지 확장하지 말 것
- 1단계에서 결제 SDK 붙이지 말 것
