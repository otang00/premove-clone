## Priority 1 — 토스페이먼츠 결제시스템 도입
### 상태
- 아직 신청 진행 중
- 승인/계정 준비 전까지는 설계와 사전 정리까지만 가능

### 목표
- 예약 생성 이후 결제 의도 생성 → 결제 승인 → 예약 확정/실패 분기까지 연결
- 현재 임시 `confirmation_pending + pending` 흐름을 실제 결제 승인 흐름으로 치환

### 선행조건
- 토스페이먼츠 신청/심사 완료
- 운영 키 발급
- 결제 성공/실패/취소/환불 처리 범위 확정

### 다음 시작점
- 승인 완료 후 결제 상태 모델과 webhook/API 설계 문서 먼저 잠금

## Priority 2 — 카카오 쉬운연결
### 상태
- 다음 단기 작업 후보

### 목표
- 회원가입/로그인 진입 장벽을 낮추는 간편 연결 경로 추가
- 기존 전화번호 기반 회원 구조와 충돌 없이 연결

### 체크포인트
- 기존 phone/email alias 회원 모델과 계정 귀속 충돌 여부 확인
- 카카오 연결 계정과 기존 회원 계정 병합 기준 잠금
- 약관/개인정보/동의 문구 정리

### 권장 순서
- 토스 신청 대기 중 병행 검토 가능
- 다만 실제 구현 전에는 귀속 정책 문서를 먼저 고정해야 함

## Priority 3 — 찜카 / IMS / 자체앱 예약 시스템 통합 연동
### 상태
- 반드시 해야 하는 중기 작업

### 목표
- 외부 예약 유입 채널과 자체 예약 시스템을 한 운영 모델로 묶기
- 예약 원천별 상태 차이를 흡수하고, 통합 운영/동기화 기준 마련

### 포함 범위
- 찜카 연동
- IMS 연동
- 자체 앱/웹 예약 연동

### 핵심 설계 포인트
- source-of-truth 결정
- 예약 상태 매핑 표준화
- 중복 예약/중복 고객/중복 차량 처리 규칙
- sync 실패 복구 규칙
- 관리자 화면에서 채널별 구분과 통합 뷰 제공 방식

### 권장 순서
- 결제 구조가 어느 정도 잠긴 뒤 본격 착수
- 먼저 상태 매핑/데이터 귀속/동기화 정책부터 문서화

## 제안 우선순위

2. 카카오 쉬운연결 정책/귀속 기준 정리
3. 외부 예약 시스템 통합 설계 착수
