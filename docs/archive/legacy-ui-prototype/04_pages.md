# 04. 페이지 설계

## A. 메인 예약 페이지
### 역할
- 서비스 첫 진입 화면
- 예약 조건 입력의 시작점
- 검색 후 차량 목록 페이지로 이동시키는 엔트리 페이지

### 필수 섹션
- 헤더
- 메인 비주얼 또는 안내 배너
- 메인 예약 박스
- 보조 신뢰 영역 또는 서비스 소개 영역
- 푸터

### 메인 예약 박스 필수 필드
- 픽업 / 딜리버리 토글
- 대여/반납 위치
- 대여/반납 일정
- 운전자 연령
- 검색 버튼

## B. 차량 목록 페이지
### 필수 섹션
- 헤더
- 안내 배너
- 검색 조건 박스
- 결과/정렬 바
- 차량 카드 리스트
- 푸터

### 차량 카드 필수 필드
- 차량 이미지
- 차량명
- 연식
- 연령 조건
- 연료
- 승차 인원
- 가격 2종 노출
- 옵션 요약

### 검색 조건 박스 필수 필드
- 픽업 / 딜리버리 토글
- 대여/반납 위치
- 대여/반납 일정
- 운전자 연령
- 검색 버튼

## C. 차량 상세/예약 페이지
### 상단 요약
- 차량 이미지
- 차량명
- 연식
- 연료
- 승차인원
- 대여/반납 시간
- 옵션 요약

### 탭 영역
- 예약 정보
- 보험/유의사항
- 업체 정보

### 보험 정보
- 보험 종류
- 추가 비용
- 보상한도
- 자차 면책금

### 운전자 정보 폼
- 이름
- 생년월일
- 휴대폰번호
- 인증번호 입력/버튼
- 대여 조건 안내문

### 차량 대여 방법
- 업체 직접 방문
- 딜리버리

### 업체 정보
- 업체명
- 주소
- 안내사항
- 지도 영역

### 결제 수단
- 카드
- 카카오페이
- 일반결제

### 약관 동의
- 약관 리스트
- 보기 버튼
- 전체 확인 동의 체크

### 우측 결제 박스
- 대여료
- 보험료
- 총 결제 금액
- 결제 버튼
