# 01. 범위 정의

## 목표
프리무브 파트너 사이트의 공개 사용자 화면을 기준으로 1단계 외형만 재현한다.

## 이번 단계에서 반드시 만드는 것
- 차량 목록 화면
- 차량 상세 화면
- 예약정보 입력 영역
- 결제 요약 박스
- 공통 헤더/푸터
- 필터/정렬 UI
- 약관동의 UI
- 지도 자리 UI

## 이번 단계에서 만들되 작동은 더미 처리할 것
- 검색 버튼
- 정렬 버튼
- 탭 전환
- 보험 선택 UI
- 결제수단 선택 UI
- 약관 보기 버튼
- 결제 버튼

## 이번 단계에서 제외
- 빈 차 계산 로직
- 예약 충돌 체크
- 문자 인증
- 실제 결제
- 카카오페이/PG 연동
- 관리자 백오피스
- 회원/로그인 체계

## 외형 구축 기준
- 사용자가 봤을 때 같은 서비스 계열로 인식될 수준
- 레이아웃, 정보 위계, 카드 구조, 폼 구조, 버튼 배치 우선
- 픽셀 단위 완전 복제보다 운영 가능한 재현을 우선
