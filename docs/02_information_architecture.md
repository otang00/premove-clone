# 02. 정보 구조

## 사용자 흐름
1. 메인 예약 페이지 진입
2. 예약 조건 입력
   - 픽업/딜리버리
   - 대여/반납 위치
   - 대여/반납 일정
   - 운전자 연령
3. 검색 실행
4. 차량 목록 페이지 이동
5. 차량 선택
6. 차량 상세/예약 페이지 이동
7. 예약정보 입력 및 결제 직전 화면 확인

## 라우트 구조
- `/` : 메인 예약 페이지
- `/cars` : 차량 목록 페이지
- `/cars/:carId` : 차량 상세/예약 페이지
- `/reservations` : 예약내역 placeholder
- `/faq` : FAQ placeholder

## URL 규칙
- 메인과 목록, 상세는 검색 조건을 query parameter로 공유한다
- 상세 페이지 진입 시 차량 식별자와 예약 조건이 함께 전달된다
- 기준 파라미터
  - `deliveryDateTime`
  - `returnDateTime`
  - `driverAge`
  - `pickupOption`
  - `order`

## 전역 레이아웃
- 상단 고정 헤더
  - 로고
  - 예약내역
  - FAQ
- 본문
- 하단 푸터
  - 약관 링크
  - 업체 정보
  - 플랫폼 고지 문구

## 메인 예약 페이지 구조
1. 상단 헤더
2. 메인 비주얼 또는 안내 배너
3. 메인 예약 박스
   - 픽업/딜리버리
   - 위치
   - 일정
   - 운전자 연령
   - 검색 버튼
4. 서비스 신뢰 문구 또는 보조 영역
5. 푸터

## 차량 목록 페이지 구조
1. 상단 배너
2. 검색 조건 박스
3. 결과 요약 영역
   - 총 차량 수
   - 정렬 버튼
4. 차량 카드 리스트
5. 상단 이동 버튼
6. 푸터

## 차량 상세/예약 페이지 구조
1. 상단 검색 조건 박스
2. 좌측 메인 컬럼
   - 차량 요약 카드
   - 탭(예약 정보 / 보험유의사항 / 업체 정보)
   - 보험 정보
   - 운전자 정보 폼
   - 차량 대여 방법
   - 지도/업체 정보
   - 결제 수단
   - 약관 동의
3. 우측 사이드 컬럼
   - 결제 정보 박스
   - 결제 버튼
4. 푸터
